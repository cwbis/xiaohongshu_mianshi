## Why

当前帖子库只提供平铺列表浏览，帖子一多之后，很难快速查看同一公司、同一岗位方向的样本。随着数据库里帖子逐步积累，用户需要一个更适合“按主题整理和回看”的管理视图，降低浏览成本并提升筛选效率。

## What Changes

- 在帖子库页增加“按公司 / 岗位分组”的浏览与管理视图。
- 支持基于帖子已有字段自动聚合分组，并展示每个分组的帖子数量与关键信息。
- 支持在分组视图中展开查看组内帖子、切换排序方式，并继续执行删除、载入草稿等现有管理动作。
- 增加帖子库分组视图与平铺视图之间的切换能力，兼顾概览和逐条浏览两种使用方式。
- 明确当帖子缺少公司或岗位信息时的分组兜底规则，避免出现无法归类的数据。

## Capabilities

### New Capabilities
- `grouped-library-browsing`: 为帖子库提供按公司、岗位等维度聚合浏览和组内管理能力。

### Modified Capabilities
- None.

## Impact

- 受影响代码：`app.js`、`router.js`、样式与帖子库渲染相关前端代码。
- API / 数据影响：默认基于现有帖子字段分组，不要求新增后端接口；如需保存用户视图偏好，可复用现有本地设置持久化能力。
- 交互影响：帖子库从单一平铺列表扩展为“概览分组 + 组内浏览 + 原有单条管理操作”的组合体验。
