# OfferScope 页面重构计划

## 目标

把当前“所有功能堆在一个页面”的形态，重构成按功能分区的多路由页面，同时保证帖子采集、帖子管理、规则分析和大模型回答仍能正常使用。

## 本轮计划

1. 增加前端路由层
   - 引入 hash 路由
   - 定义页面元信息和默认路由
   - 实现导航高亮和非法路由回退

2. 拆分页面布局
   - 采集页：目标配置、单条录入、自动填充、批量导入
   - 帖子库页：列表、删除、载入草稿、导入导出
   - 分析页：关键词、问题、流程、Markdown 报告
   - 大模型页：模型配置、问题回答、总结

3. 保持共享状态
   - `posts` 继续用 `localStorage`
   - `llmConfig` 继续持久化
   - `llmResult` 保留在前端状态中

4. 清理主脚本结构
   - 把页面渲染按页面函数组织
   - 保留现有桥接和分析逻辑
   - 减少“一个大页面绑定所有 DOM”的耦合

5. 完成验证
   - 路由解析测试
   - 分析核心 smoke test
   - 主脚本语法检查
   - 页面交互链路手工检查点整理

## 验收标准

- 页面可通过导航切换到四个功能页
- 刷新页面后仍能回到对应 hash 路由
- 示例帖子可被导入并在帖子库展示
- 分析页能显示关键词、问题和报告
- 大模型页能读取配置并触发请求
- 报告复制按钮仍可使用
