## 1. 训练模式边界与目标建模
- [x] 1.1 在面试 Agent 中明确区分“问答模式”和“训练模式”，定义训练模式的入口与最小交互边界。
- [x] 1.2 为训练模式定义轻量 `Goal` 模型，至少覆盖目标公司、目标岗位、训练模式和专题偏好。
- [x] 1.3 明确训练模式下的专题选择输入来源，包括本地高频题、专题题库和知识点模板。

## 2. 训练状态机与核心对象
- [x] 2.1 为训练模式落定最小状态机：`GOAL_SET -> TOPIC_PICK -> ASK -> WAIT_ANSWER -> EVALUATE -> DECIDE_NEXT`。
- [x] 2.2 定义 `Goal`、`TopicProgress`、`Attempt`、`ReviewSummary` 四个核心对象的职责和字段边界。
- [x] 2.3 明确 `DECIDE_NEXT` 允许的最小动作集合：`FOLLOW_UP`、`NEXT_QUESTION`、`SWITCH_TOPIC`、`SUMMARY`。

## 3. 回答评估与决策规则
- [x] 3.1 为 `EVALUATE` 设计结构化输出格式，至少包括 `answerQuality`、`keyPointCoverage`、`factualRisk`、`expressionClarity`、`shouldFollowUp`、`weaknessTags` 和 `confidence`。
- [x] 3.2 明确给用户展示的评估反馈结构，至少包括 `strengths`、`gaps` 和 `suggestion`。
- [x] 3.3 为 `DECIDE_NEXT` 制定第一版规则，覆盖回答差 / 一般 / 较好时的追问、切题、切专题和总结逻辑。
- [x] 3.4 约束追问次数、单轮题数和专题切换阈值，优先保证训练节奏稳定与可解释性。

## 4. 持久化与后端接口演进
- [x] 4.1 设计训练会话、答题尝试、评估结果和阶段总结的本地持久化边界。
- [x] 4.2 评估并扩展现有 Agent 会话模型，为训练模式增加独立的模式、状态和结构化字段。
- [x] 4.3 扩展 Agent API，支持训练目标设置、主动出题、提交作答、获取评估和推进下一步。

## 5. 前端训练交互规划
- [x] 5.1 设计 `AgentPage` 中训练模式的入口、目标输入和训练状态展示区域。
- [x] 5.2 规划训练模式下的主动出题展示、用户作答输入和评估反馈呈现方式。
- [x] 5.3 规划训练过程中“继续追问 / 下一题 / 切专题 / 直接总结”等动作的前端承载方式。
- [x] 5.4 设计阶段性小结和训练复盘的展示方式，保证用户能看到当前专题表现和下一步建议。

## 6. 风险控制与验证
- [x] 6.1 明确训练模式与现有问答模式的隔离策略，避免新模式干扰已有单轮问答体验。
- [x] 6.2 为 `EVALUATE` 和 `DECIDE_NEXT` 设计最小验证方案，确认输出稳定性和决策可解释性。
- [x] 6.3 确定 MVP 的回退策略：如果训练闭环体验不稳定，可退回“主动出题 + 简单反馈”的弱闭环模式。
- [x] 6.4 在实现前复核开放问题，包括训练模式数量、`TopicProgress` 的持久化方式和 `ReviewSummary` 的触发节奏。
