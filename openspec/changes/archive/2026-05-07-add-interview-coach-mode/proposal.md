## Why

当前“面试 Agent”已经具备问题分类、本地检索、结构化回答、追问和会话历史等能力，但整体仍然更接近“带知识库的问答助手”，而不是“围绕面试目标持续推进训练的教练型 Agent”。

对于求职准备场景，用户真正需要的往往不是单轮问答本身，而是一个能围绕目标公司、岗位和专题优先级持续组织训练的系统：它能主动出题、评估用户回答、决定下一步是追问还是切题，并在若干轮之后给出复盘总结。当前系统缺少的不是更强的单轮回答，而是“训练状态”“评估信号”和“下一步决策”这三层能力。

因此，需要把现有面试 Agent 从“响应式问答”升级为“半主动、可控、可复盘的训练模式”，先形成一个最小可行的教练型闭环。

## What Changes

- 在现有面试 Agent 基础上新增“训练模式 / 教练模式”，让系统可以围绕用户目标主动组织练习，而不是只在用户提问后回答。
- 引入轻量目标建模，至少覆盖目标公司、目标岗位、训练模式和专题偏好，用于驱动训练上下文。
- 新增训练流程状态机，最小闭环包括：`GOAL_SET -> TOPIC_PICK -> ASK -> WAIT_ANSWER -> EVALUATE -> DECIDE_NEXT`。
- 支持 Agent 基于本地高频题、专题题库和知识点主动出题，而不是完全依赖用户手工输入问题。
- 新增结构化回答评估输出，重点覆盖：回答质量、关键点覆盖、事实风险、表达清晰度、追问建议和薄弱点标签。
- 新增“下一步动作”决策逻辑，用于在 `FOLLOW_UP`、`NEXT_QUESTION`、`SWITCH_TOPIC` 和 `SUMMARY` 之间做稳定、可解释的推进。
- 在一次训练会话中记录题目、用户作答、评估结果、追问次数和阶段总结，为后续薄弱点画像和长期复盘打基础。
- 在前端面试 Agent 页面中增加训练模式入口、主动出题展示、作答输入、评估反馈和阶段总结展示。

## Capabilities

### New Capabilities
- `interview-coaching-mode`: 为面试 Agent 提供目标驱动、主动出题、回答评估、追问推进和阶段复盘能力。

### Modified Capabilities
- `interview-knowledge-agent`: 从“分类 + 检索 + 结构化回答”扩展到“可进入训练闭环的教练型交互”。
- `react-frontend-shell`: 面试 Agent 页面从问答界面扩展为“问答模式 + 训练模式”的双模式交互。
- `local-persistent-store`: 增加训练会话、答题尝试、评估结果和阶段总结等持久化需求。

## Impact

- 受影响代码：
  - 后端 `backend/api/agent.py`
  - 训练编排、评估和决策相关服务
  - 训练状态与记录相关 repository / schema
  - React 端 `AgentPage` 和相关 API client / 状态管理
- API / 数据影响：
  - 需要扩展 Agent 相关请求与响应结构
  - 需要为训练会话、答题尝试、评估结果和总结增加本地持久化模型
  - 可能需要在现有会话模型之外增加更明确的训练状态字段
- 交互影响：
  - 面试 Agent 不再只是“输入问题 -> 生成回答”
  - 用户将可以进入训练模式，由 Agent 主动出题、给出反馈并推动下一步
- 风险与约束：
  - 第一版应保持“半主动、可控”的边界，避免一开始做成过重的全自治 Agent
  - `EVALUATE` 与 `DECIDE_NEXT` 必须解耦：前者输出结构化训练信号，后者结合上下文做最终动作决策
  - MVP 优先追求节奏稳定与可解释性，而不是追求高度自由的智能决策
